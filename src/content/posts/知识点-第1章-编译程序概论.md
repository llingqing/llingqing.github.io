## 一、翻译程序、编译程序、解释程序

### 🟡 翻译程序（Translator）
把一种语言程序（**源语言程序**）转换成另一种功能**等价**的语言程序（**目标语言程序**）的程序。它是一个总称，编译程序和解释程序都属于翻译程序。

### 🟡 编译程序（Compiler）
一种特殊的翻译程序，其**源程序是高级语言**，**目标程序是低级语言**（汇编语言或机器语言）。
- 特点：**一次性把整个源程序翻译成目标程序**，之后可反复执行目标程序。
- 优点：目标程序执行**速度快**。
- 例子：C、TC 等高级语言编译系统。

### 🟡 解释程序（Interpreter）
也是翻译程序，但与编译程序的根本区别在于：
- **逐句翻译、边翻译边执行**，通常**不生成目标程序**。
- 优点：便于调试、交互性好、可移植。
- 缺点：执行**速度慢**（每次执行都要重新翻译）。
- 例子：BASIC、早期 JAVA 的解释执行等。

> **编译 vs 解释 一句话记忆**：编译"先翻完再跑、生成目标程序、快"；解释"边翻边跑、不生成目标程序、慢"。

### 🟢 高级语言的完整处理过程
```
源程序 ──预处理程序──▶ 源程序 ──编译程序──▶ 汇编语言/目标语言
      ──汇编程序──▶ 可重定位目标文件（机器码）
      ──装配/连接程序──▶ 绝对机器代码（可执行）
```

## 二、编译过程的六个阶段（🔴 必背核心）

`词法分析 → 语法分析 → 语义分析 → 中间代码生成 → 代码优化 → 目标代码生成`

配合两个**贯穿始终**的辅助模块：**表格（符号表）管理程序**、**出错处理程序**。

### 1. 词法分析（Lexical Analysis）
- **任务**：扫描源程序字符串，识别出一个个**单词符号**（token）。
- **输入**：源程序（字符流）。
- **输出**：单词符号序列（二元组）。
- 例：`main()` → 保留字 `main`、左括号 `(`、右括号 `)`、左花括号 `{`……

### 2. 语法分析（Syntax Analysis）
- **任务**：根据语法规则，把单词序列组合成**语法短语**（如表达式、语句），判断语法是否正确。
- **输入**：单词符号序列。
- **输出**：语法树 / 语法单位。

### 3. 语义分析（Semantic Analysis）
- **任务**：对语法单位进行**语义审查**——类型检查、类型匹配、收集类型信息，报告语义错误。

### 4. 中间代码生成（Intermediate Code Generation）
- **任务**：把源程序转换成一种**结构简单、含义明确、易生成、易译成目标代码**的中间形式。
- **常用形式**：**四元式** `(算符, 运算对象1, 运算对象2, 结果)`；此外还有三元式、逆波兰式等。

### 5. 代码优化（Code Optimization）
- **任务**：对中间代码或目标代码作等价变换，使生成代码**运行更快、占用空间更小**。

### 6. 目标代码生成（Target Code Generation）
- **任务**：把中间代码/语义分析结果翻译成**特定机器**上的绝对指令、可重定位指令或汇编代码。

## 三、编译程序的结构与组合

### 🟡 前端（Front End） / 后端（Back End）
- **前端**：与**源语言**有关、与**目标机无关**的部分——对应第 1~4 阶段（词法、语法、语义、中间代码生成）。
- **后端**：与**目标机**有关的部分——对应第 6 阶段（目标代码生成）。
- **代码优化**（第 5 阶段）：置于前端或后端**均可**。

### 🟡 前后端组合的两大用途
1. **同一前端 + 不同后端** → 同一源语言在**不同机器**上的编译程序（换机器只需换后端）。
2. **不同前端生成同一中间语言 + 共用后端** → 同一机器上**多种语言**的编译程序（加语言只需加前端）。

### 🟡 遍 / 趟（Pass）
- **定义**：对源程序或其中间结果**从头到尾扫描一次**，称为一遍。每遍完成一个或几个阶段的工作。
- 一个编译程序可由**一遍或多遍**完成，分遍的主要依据是**源语言与目标机的特征**。
- **多遍优缺点**：
  - 优点：结构清晰、功能单一、**节省内存**。
  - 缺点：增加 I/O 次数、**降低效率**。

## ⚠️ 易错点汇总
- 把**汇编语言**翻译成机器码的是 **汇编器（Assembler）**，不是编译器。
- 编译方式**生成**目标程序、执行快；解释方式**不生成**目标程序、便于调试但慢。
- 中间代码最常见形式是**四元式**，格式 `(算符, 对象1, 对象2, 结果)`。
- 符号表管理和出错处理**不是某一个阶段**，而是贯穿编译全过程的辅助模块。
